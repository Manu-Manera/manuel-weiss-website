# Demo 1: Project Manager Perspektive (Phase 2, Woche 5-6)

## Zielgruppe
- Interne Präsentation für Marc/Aayushi
- Später: Kunden-Demos

## Dauer
- 10-15 Minuten

## Benötigte Vorbereitung
- [ ] Trial Environment Zugang
- [ ] Testprojekt vorbereitet
- [ ] Bildschirm-Sharing eingerichtet

---

## Demo-Script

### 1. Einleitung (1 Min)

```
"Ich zeige Ihnen heute Tempus Resource aus der Perspektive eines 
Project Managers. Als PM ist mein Hauptziel, mein Projekt mit den 
richtigen Ressourcen zur richtigen Zeit zu besetzen."
```

### 2. UI Navigation (2 Min)

**Zeigen:**
- [ ] Login und Dashboard
- [ ] Hauptnavigation erklären
- [ ] PM Tile lokalisieren

**Sagen:**
```
"Nach dem Login sehen wir das Dashboard mit den wichtigsten 
Informationen auf einen Blick. Als Project Manager arbeite ich 
hauptsächlich im PM-Bereich hier..."
```

### 3. Projekt erstellen (3 Min)

**Schritte:**
1. [ ] "Neues Projekt" klicken
2. [ ] Projektname eingeben
3. [ ] Attributes konfigurieren:
   - Status
   - Start-/Enddatum
   - Priorität
   - Budget (falls vorhanden)
4. [ ] Projekt speichern

**Sagen:**
```
"Um ein neues Projekt anzulegen, klicke ich hier auf 'Neues Projekt'. 
Jetzt kann ich die wichtigsten Attribute definieren - den Namen, 
den Zeitraum, die Priorität. Diese Attribute sind übrigens 
kundenspezifisch konfigurierbar."
```

### 4. Ressourcenbedarf definieren (3 Min)

**Schritte:**
1. [ ] Demand/Allocation hinzufügen
2. [ ] Rolle/Skill definieren
3. [ ] Zeitraum festlegen
4. [ ] Aufwand eingeben (FTE oder Stunden)

**Sagen:**
```
"Jetzt definiere ich meinen Ressourcenbedarf. Ich brauche zum 
Beispiel einen Senior Developer für 3 Monate mit 50% Auslastung. 
Das System zeigt mir sofort, ob generell Kapazität verfügbar ist."
```

### 5. Resource Request erstellen (2 Min)

**Schritte:**
1. [ ] Request erstellen
2. [ ] Details ausfüllen
3. [ ] An Resource Manager senden

**Sagen:**
```
"Um eine konkrete Person zu bekommen, erstelle ich einen Resource 
Request. Dieser geht an den Resource Manager, der dann prüft, 
wer verfügbar ist und die passenden Skills hat."
```

### 6. PM Views demonstrieren (3 Min)

**Zeigen:**
- [ ] Grid View: Tabellarische Übersicht
- [ ] Gantt View: Zeitliche Darstellung
- [ ] Kanban View: Status-basierte Ansicht

**Sagen:**
```
"Tempus bietet verschiedene Ansichten für unterschiedliche 
Bedürfnisse. Die Grid-Ansicht für Details, Gantt für die 
Timeline, und Kanban für agile Workflows."
```

### 7. Zusammenfassung (1 Min)

```
"Zusammenfassend: Als Project Manager kann ich in Tempus meine 
Projekte anlegen, den Ressourcenbedarf definieren, Anfragen an 
das Resource Management stellen und den Fortschritt in 
verschiedenen Views verfolgen. Haben Sie Fragen?"
```

---

## Häufige Fragen vorbereiten

| Frage | Antwort |
|-------|---------|
| Kann ich mehrere Projekte gleichzeitig sehen? | Ja, über die Portfolio-Ansicht oder Filter |
| Was passiert, wenn mein Request abgelehnt wird? | Sie können nachverhandeln oder alternative Ressourcen anfragen |
| Kann ich sehen, warum eine Ressource nicht verfügbar ist? | Ja, über das Ressourcenprofil (mit entsprechenden Rechten) |

---

## Selbst-Check nach der Demo

- [ ] Alle Schritte flüssig durchgeführt?
- [ ] Fragen souverän beantwortet?
- [ ] Zeitrahmen eingehalten?
- [ ] Feedback notiert?

# Demo 2: Resource Manager Perspektive (Phase 2, Woche 7-8)

## Zielgruppe
- Interne Präsentation für Marc/Aayushi
- Später: Kunden-Demos

## Dauer
- 10-15 Minuten

## Benötigte Vorbereitung
- [ ] Trial Environment Zugang
- [ ] Offene Resource Requests vorhanden
- [ ] Testressourcen mit Profilen

---

## Demo-Script

### 1. Einleitung (1 Min)

```
"Jetzt wechseln wir die Perspektive zum Resource Manager. Meine 
Aufgabe ist es, die Ressourcen optimal auf die Projekte zu 
verteilen und sicherzustellen, dass niemand über- oder 
unterausgelastet ist."
```

### 2. RM Dashboard (2 Min)

**Zeigen:**
- [ ] RM Tile/Bereich
- [ ] Übersicht offener Requests
- [ ] Kapazitätsübersicht

**Sagen:**
```
"Im Resource Manager Bereich sehe ich sofort alle offenen 
Anfragen und habe einen Überblick über die Gesamtkapazität 
meines Teams."
```

### 3. Resource Request beantworten (4 Min)

**Schritte:**
1. [ ] Offenen Request öffnen
2. [ ] Anforderungen prüfen (Skill, Zeitraum, Aufwand)
3. [ ] Verfügbare Ressourcen suchen
4. [ ] Passende Ressource auswählen
5. [ ] Request beantworten (Approve/Negotiate)

**Sagen:**
```
"Hier sehe ich einen Request für einen Senior Developer. Ich 
prüfe jetzt, wer in meinem Team diese Skills hat und im 
gewünschten Zeitraum verfügbar ist..."
```

### 4. Individual Resource Profile (3 Min)

**Zeigen:**
- [ ] Ressourcenprofil öffnen
- [ ] Skills und Qualifikationen
- [ ] Aktuelle Zuweisungen
- [ ] Verfügbarkeit über Zeit

**Sagen:**
```
"Das Ressourcenprofil zeigt mir alle wichtigen Informationen: 
Skills, aktuelle Projekte, Auslastung. So kann ich fundierte 
Entscheidungen treffen."
```

### 5. Net Availability (3 Min)

**Zeigen:**
- [ ] Net Availability View öffnen
- [ ] Zeitliche Darstellung der Verfügbarkeit
- [ ] Engpässe identifizieren

**Sagen:**
```
"Die Net Availability zeigt mir die verbleibende Kapazität nach 
Abzug aller Zuweisungen. Rot bedeutet Überauslastung, hier 
muss ich handeln."
```

### 6. Grid View für RMs (2 Min)

**Zeigen:**
- [ ] RM-spezifische Grid-Ansicht
- [ ] Filter und Gruppierung
- [ ] Bulk-Aktionen

**Sagen:**
```
"In der Grid-Ansicht kann ich schnell filtern und sortieren, 
zum Beispiel alle Ressourcen mit einem bestimmten Skill oder 
alle mit freier Kapazität."
```

### 7. Zusammenfassung (1 Min)

```
"Als Resource Manager habe ich in Tempus alle Werkzeuge, um 
Anfragen effizient zu bearbeiten, die Auslastung zu optimieren 
und Engpässe frühzeitig zu erkennen. Fragen?"
```

---

## Häufige Fragen vorbereiten

| Frage | Antwort |
|-------|---------|
| Was, wenn keine passende Ressource verfügbar ist? | Negotiate mit PM, Alternativen vorschlagen, externe Ressourcen |
| Kann ich Ressourcen zwischen Projekten verschieben? | Ja, über BPAFG oder direkte Bearbeitung |
| Wie sehe ich historische Auslastung? | Über Reports und Snapshots |

---

## Selbst-Check nach der Demo

- [ ] Alle Schritte flüssig durchgeführt?
- [ ] Unterschied zu PM-Perspektive klar gemacht?
- [ ] Fragen souverän beantwortet?
- [ ] Feedback notiert?

# Demo 3: Reporting & Portfolio Management (Phase 3, Woche 9-10)

## Zielgruppe
- Interne Präsentation für Marc/Aayushi
- Später: Management-Demos bei Kunden

## Dauer
- 10-15 Minuten

## Benötigte Vorbereitung
- [ ] Trial Environment mit Daten
- [ ] Beispiel-Reports vorbereitet
- [ ] What-If Szenario angelegt

---

## Demo-Script

### 1. Einleitung (1 Min)

```
"In dieser Demo zeige ich die Reporting- und Analyse-Funktionen 
von Tempus Resource. Diese sind besonders wertvoll für 
strategische Entscheidungen und Management-Reporting."
```

### 2. RAR Report (2 Min)

**Zeigen:**
- [ ] RAR öffnen
- [ ] Filter setzen
- [ ] Ergebnisse interpretieren

**Sagen:**
```
"Der Resource Allocation Report gibt einen schnellen Überblick 
über alle Zuweisungen. Ich kann nach Projekt, Ressource, 
Zeitraum oder Skills filtern."
```

### 3. RAR2 Report (2 Min)

**Zeigen:**
- [ ] RAR2 öffnen
- [ ] Erweiterte Optionen zeigen
- [ ] Unterschied zu RAR erklären

**Sagen:**
```
"RAR2 bietet erweiterte Möglichkeiten wie zusätzliche 
Gruppierungen und detailliertere Filteroptionen."
```

### 4. Pivot Grid (3 Min)

**Zeigen:**
- [ ] Pivot Grid öffnen
- [ ] Dimensionen per Drag & Drop ändern
- [ ] Aggregationen zeigen

**Sagen:**
```
"Das Pivot Grid ist besonders flexibel. Ich kann Dimensionen 
frei anordnen und so verschiedene Perspektiven auf die 
gleichen Daten bekommen."
```

### 5. Portfolio Planner (3 Min)

**Zeigen:**
- [ ] Portfolio Planner öffnen
- [ ] Projektübersicht
- [ ] Ressourcen-Engpässe visualisieren
- [ ] Priorisierung demonstrieren

**Sagen:**
```
"Der Portfolio Planner ist das strategische Werkzeug für 
Führungskräfte. Hier sehe ich alle Projekte, deren 
Ressourcenbedarf und kann Prioritäten setzen."
```

### 6. What-If Scenario (3 Min)

**Zeigen:**
- [ ] Szenario erstellen/öffnen
- [ ] Änderung simulieren (z.B. Projekt verschieben)
- [ ] Auswirkungen analysieren
- [ ] Mit Baseline vergleichen

**Sagen:**
```
"Mit What-If Szenarien kann ich Änderungen simulieren, ohne 
die Produktivdaten zu beeinflussen. Was passiert, wenn wir 
Projekt X um einen Monat verschieben? Das sehen wir hier..."
```

### 7. Dashboard (2 Min)

**Zeigen:**
- [ ] Dashboard öffnen
- [ ] Verschiedene Widgets erklären
- [ ] Anpassungsmöglichkeiten

**Sagen:**
```
"Dashboards fassen die wichtigsten KPIs zusammen. Diese sind 
individuell konfigurierbar je nach Rolle und Bedarf."
```

### 8. Zusammenfassung (1 Min)

```
"Mit den Reporting-Funktionen von Tempus haben Sie alle 
Informationen für operative und strategische Entscheidungen. 
Von Detail-Reports bis zur Portfolio-Übersicht und 
Szenario-Planung. Fragen?"
```

---

## Häufige Fragen vorbereiten

| Frage | Antwort |
|-------|---------|
| Kann ich Reports exportieren? | Ja, Excel, PDF, etc. |
| Wie oft werden Daten aktualisiert? | Echtzeit oder nach Sync-Intervall |
| Kann ich eigene Reports erstellen? | Ja, über View Management und Pivot Grid |
| Wer sieht welche Daten? | Über Berechtigungen konfigurierbar |
