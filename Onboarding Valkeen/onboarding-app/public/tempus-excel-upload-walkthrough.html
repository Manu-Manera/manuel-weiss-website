02
Das große Ganze

Tempus ist anfangs ein leeres Blatt

Wir befüllen Tempus über standardisierte Excel-Vorlagen. Jede Vorlage steht für einen Datentyp – und diese Typen bauen logisch aufeinander auf.

  • Stammdaten zuerst: Felder (Attribute), Ressourcen und Skills sind das Fundament.
  • Dann die Container: Projekte – sie nutzen die Attribute aus Schritt 1.
  • Dann die Verknüpfungen: Zuweisungen verbinden Ressource + Projekt + Aufwand.
  • Zuletzt die Details: Finanzen, Raten, Admin-Zeiten, Team-Mitglieder.
💡 Kernidee
Eine Zuweisung kann nur entstehen, wenn die Ressource und das Projekt bereits existieren. Deshalb ist die Reihenfolge keine Formalität, sondern Voraussetzung.
Felder→ Ressourcen→ Projekte→ Zuweisungen→ Details
03
Der Upload-Ort

Wo der Excel-Import passiert

Alle Vorlagen laufen über denselben Ort in Tempus – nur die Datei wechselt.

  1. Anmelden & navigieren Admin Settings > Data Synchronization > Excel
  2. Datei wählen – Choose File klicken oder die .xlsx auf die Seite ziehen.
  3. Synchronisieren – Button Synchronize startet den Import.
⚠️ Zwei Regeln, die oft vergessen werden
  • Die auszufüllende Vorlage muss die erste Registerkarte (erstes Tab) der Datei sein.
  • Dateiformat ist immer .xlsx – pro Synchronisation eine Vorlage / ein Datentyp.
✅ Empfehlung
Erst in der Sandbox testen (gleicher Pfad), dann in Produktion. So sehen Sie Fehler, bevor echte Daten betroffen sind.

Überblick

Die 10 Import-Dateien

Jede Datei hat ein festes Arbeitsblatt (Sheet) und einen klaren Zweck. Farbe = Ebene der Abhängigkeit (blau = Fundament, lila = baut auf Fundament auf, gold = Detail-/Finanzdaten).

Attributes

Eigene Felder (Custom Fields) für Projekte, Ressourcen, Zuweisungen, Milestones, Finanzen – inkl. Auswahllisten.

Resources

Unabhängig – jederzeit importierbar

Projektliste mit Start/Ende, Dataset Preference (Allocation/Demand), Priorität, Phase, Benefit.

05
Zusammenhänge

Wie die Dateien voneinander abhängen

Drei Ebenen. Was unten steht, setzt voraus, dass die Ebene darüber bereits in Tempus existiert.

⬤ Ebene 1 · Fundament (keine Voraussetzungen)
1 Attributes Felder & Auswahllisten
2 Resources Ressourcen-Pool
6 Skills Skill-Katalog
⬤ Ebene 2 · Baut auf dem Fundament auf
3 Projects ← Attribute (1)
8 Advanced Rates ← Ressourcen (2)
10 Team Resources ← Ressourcen (2)
5 Admin Times ← Ressourcen (2) · Attr. (1)
⬤ Ebene 3 · Verknüpft Projekte & Ressourcen
4 Allocation & Tasks ← Ressourcen (2) + Projekte (3)
9 Financials ← Projekte (3) · Kategorien (1)
7 Sheets ← Projekte/Ressourcen + Template
🚫 Typischer Stolperstein
Eine Zuweisung (4) auf „Projekt X / Max M." schlägt fehl, wenn Projekt X oder Max M. noch nicht existieren. Referenzierte Objekte müssen vorher angelegt sein.

Reihenfolge

Die offizielle Nummerierung 1 → 10 bildet genau die Abhängigkeiten ab. Wer der Reihe folgt, vermeidet die meisten Fehler automatisch.

2ResourcesDer Ressourcen-Pool – Grundlage für Zuweisungen, Raten, Team, Admin-Zeiten.keine

✅ Faustregel
Stammdaten → Container → Verknüpfungen → Details. Innerhalb dieser Logik sind kleine Umstellungen ok (z. B. Skills früh), die drei Ebenen aber nie überspringen.
07
Erkennung

Wie Tempus Zeilen wiedererkennt

Beim erneuten Import entscheidet ein Schlüssel, ob ein Datensatz neu angelegt oder aktualisiert wird. Das ist die Basis für saubere Updates.

DateiSchlüssel für Wiedererkennung
Attributes (1)Custom Field Name + Entity Type
Resources (2)Resource Name (oder API External Id, falls so konfiguriert)
Projects (3)Project Name (oder API External Id)
Allocation & Tasks (4)Project + Resource + Task + Plan Type + Allocation Type
Skills (6)Skill Name
💡 Praxis
Halten Sie Namen über alle Dateien hinweg identisch (z. B. „Max Müller" überall gleich geschrieben). Schon ein Tippfehler erzeugt versehentlich einen zweiten Datensatz.
⚙️ Projekt-Level-Zuweisung
Soll auf Projektebene (ohne echten Task) geplant werden, nutzen Sie den eingebauten Task-Namen Generic in Spalte Task.
08
Steuerung pro Zeile

Import Behavior: Merge · Overwrite · Skip

Jede Zeile trägt die Spalte Import Behavior. Sie bestimmt, was Tempus mit bestehenden Daten macht.

WertWirkungEmpfehlung
MergeLegt neue Datensätze an und aktualisiert bestehende. In Tempus gepflegte Werte bleiben erhalten.Standard für fast alles
OverwriteErsetzt einen bestehenden gleichnamigen Datensatz vollständig – bisherige Werte gehen verloren.Vorsicht nur bewusst
SkipÜberspringt die komplette Zeile – nichts wird importiert.selektiv
🚫 Datenverlust-Risiko
Overwrite entfernt und ersetzt. Bei Attributen gehen damit auch zugeordnete Werte verloren. Im Zweifel immer Merge.
09
Zeitverlauf

Zeit-phasierte Spalten (Datum-Spalten)

Einige Dateien haben hinter den festen Spalten Datums-Spalten – je eine pro Periode. Der Spaltenkopf ist das Datum, der Wert die Menge in dieser Periode.

DateiRasterEinheit
Resources (2)MonatStunden (Kapazität)
Allocation (4)Projekt/Jahr/Quartal/Monat/Woche/TagHours oder FTE
Admin Times (5)WocheStunden / FTE
Financials (9)Monat/Quartal/JahrKosten
Team Resources (10)WocheFTE (Inclusion %)
💡 Time Period = Spaltenköpfe
Der Wert in Time Period muss zu den Datums-Spalten passen. Beispiel: bei Month ein beliebiges Datum im Monat; bei Week immer der Montag der Woche.
✅ Hours vs. FTE
Hours = Aufwand, der über die Arbeitstage der Periode verteilt wird.
FTE = Kapazitätsanteil, der über die Arbeitstage repliziert wird.
⚠️ Tagesraster
Bei Day müssen Datteln lückenlos sein (inkl. Sa/So) – Werte an Wochenenden werden ignoriert.
10
Live-Durchlauf

Schritt-für-Schritt: Ein sauberer Import

So gehen wir gemeinsam einen vollständigen Erst-Import durch.

  1. Vorbereiten Pro Datentyp eine .xlsx, Vorlage als erstes Tab, Namen einheitlich.
  2. Schritt 1 – Attributes Erst die Felder & Auswahllisten anlegen (sonst fehlen sie später).
  3. Schritt 2 – Resources Ressourcen-Pool inkl. Kapazität importieren.
  4. Schritt 3 – Projects Projekte anlegen; Attribute aus Schritt 1 werden befüllt.
  5. Schritt 4 – Allocation & Tasks Zuweisungen mit geplanten Werten – jetzt existieren alle Referenzen.
  6. Schritte 5–10 – Details Admin-Zeiten, Skills, Sheets, Raten, Finanzen, Team-Mitglieder ergänzen.
  7. Prüfen Nach jedem Synchronize die Rückmeldung kontrollieren und Stichproben in Tempus ansehen.
✅ Pro Schritt einzeln synchronisieren
Jede Datei separat hochladen und prüfen – so ist bei einem Fehler sofort klar, welcher Datentyp betroffen ist.

12
Zum Mitnehmen

Checkliste vor jedem Import

  • Datentyp klar → richtige Vorlage gewählt
  • Vorlage ist das erste Tab, Format .xlsx
  • Original-Spaltenüberschriften unverändert
  • Pflichtfelder gefüllt (mind. der Name)
  • Namen über alle Dateien identisch
  • Reihenfolge 1 → 10 eingehalten
  • Import Behavior bewusst gesetzt (i. d. R. Merge)
  • Datums-Spalten passen zu Time Period
  • Auswahlwerte existieren in den Attributen
  • Erst Sandbox, dann Produktion
👉 Nächste Schritte
Gemeinsamer Test-Import in der Sandbox mit Ihren echten Dateien – wir gehen Schritt 1–4 live durch und klären offene Punkte direkt am Bildschirm.
