# How to Use This Guide

:::roles PM/SM | Blended PM/SM + PPM
:::

This guide is built for **Planners** at Knauf — Project Managers (PM), Service Managers (SM), and the Blended role (PM/SM + PPM). It walks you through the two workflows you will use every day:

1. **Create a project** — set it up so that the right people, dates and money land in Tempus from day one.
2. **Find an existing project and edit it** — the same set of pages, just in update mode.

Every workflow ends in the same three pillars. Master them once, and the rest is repetition:

| Pillar | What it answers | Where you do it |
|---|---|---|
| **Attributes** | *What* is this project? (Owner, Phase, Business Unit, Dates, Tags …) | Attributes page |
| **Resources** | *Who* and *how much*? (named people, generic demand, time grain) | Single Project Allocation grid |
| **Financials** | *What does it cost?* (Hourly Rate, Cost Plan, Budget, Actuals) | Financials page |

> Tip: Whenever a step says "do the same thing later when editing", you can jump straight to the relevant pillar chapter — chapters 5–7 are written so they work for both Create and Edit.

## Roles in This Guide

| Role | What you will plan | In this guide |
|---|---|---|
| **PM** — Project Manager | Standard projects (Intake → Active → Closed) | Full scope |
| **SM** — Service Manager | Services / non-project activities (Run, Admin, Change) | Full scope; "Service" vs. "Project" differences flagged inline |
| **Blended PM/SM + PPM** | Both, plus Portfolio overview | Full scope; PPM cross-references at end of each chapter |

Out of scope here: **Resource Manager (RM)** workflows (Bulk Project Allocation, Resource Replace, Heatmap administration). A dedicated RM guide will follow.

## Time Grain Defaults — set these once

Time grain and capacity unit are the two settings that change how every grid in Tempus looks. The Knauf defaults are:

| Setting | Knauf default | Where to change |
|---|---|---|
| **Time granularity** | Month | SPA grid → ribbon above the timeframe |
| **Capacity unit** | Hours (PM/SM views also show Mandays) | SPA grid → ribbon above the timeframe |
| **Working hours / day** | 8 h | Configured globally (admin-only) |
| **Working hours / month** | ~168 h (varies per German public-holiday calendar) | Configured globally (admin-only) |

> Note: If you change time grain or capacity unit on the SPA grid, the change is **per view, per user** — it does not affect anyone else.
