Überblick: Resource Request Workflow

⏱ ca. 5 Minuten

📚 Theorie: Ablauf

Der Resource Request Workflow ist in Tempus eingebaut:

  1. PM erstellt Resource Request (Project Management → Checkout → «Type a new resource»)
  2. Status = Pending
  3. RM erhält Benachrichtigung (wenn Subscriptions aktiv)
  4. RM öffnet Resource Requests → wählt Approve / Replace / Reject
  5. Status wechselt zu Approved oder Rejected

Voraussetzung: Jede Ressource braucht einen zugewiesenen Resource Manager. Der RM muss Is Resource Manager = true haben.

Wo findest du was? Resource Requests: Tile «Resource Requests» auf der Homepage (rotes Badge = Anzahl Pending). Status: In der Liste filterbar (Pending, Approved, Rejected, …).

Rollen & Attribute

⏱ ca. 5 Minuten

📚 Theorie: Benötigte Attribute

AttributEntitätWo in TempusBedeutung
Resource ManagersResourceAttributes & IdentityWer managed diese Ressource? (z.B. User)
Is Resource ManagerResourceAttributes & IdentityIst diese Person ein RM? (Ja/Nein)
Require Resource Manager ApprovalResourceAttributes & IdentityMuss RM Assignments genehmigen? (Ja)
Project ManagerProjektProjekt-AttributeWer ist PM für dieses Projekt?

Hinweis: Attribute-Namen können je nach Tenant variieren (z.B. «SHS IT Attributes»).

Wo findest du die Attribute? Resource: Resource Management → Ressource öffnen (Klick auf Name) → Tab «Attributes & Identity» → Sektion mit Custom Fields. Projekt: Project Management → Projekt öffnen → Attributes.

User als RM & PM einrichten (UI)

⏱ ca. 10 Minuten

🛠 Schritt 1: User anlegen

  1. Resource Management → Plus (oben links)
  2. Name: User → Create
  3. Capacity Start/End Date, Feiertagskalender setzen

🛠 Schritt 2: User als Resource Manager markieren

  1. User öffnen (Klick auf Name)
  2. Tab Attributes & Identity
  3. Is Resource Manager = Ja
  4. Speichern

🛠 Schritt 3: Stern & Co Ressourcen → User als RM

  1. Für jede Ressource (Brad Pitt, Meryl Streep, Taylor Swift, …):
  2. Ressource öffnen → Attributes & Identity
  3. Resource Managers = User
  4. Require Resource Manager Approval = Ja
  5. Speichern

🛠 Schritt 4: Stern & Co Projekte → User als PM

  1. Falls das Attribut «Project Manager» existiert:
  2. Projekt öffnen (Blockbuster App 2025, Cloud Migration, …)
  3. Project Manager = User
  4. Speichern

🛠 Schritt 5: Subscriptions für User

  1. Als User einloggen
  2. Username (oben rechts) → Subscriptions
  3. Tab Resource Request → Resource request management → Submitted aktivieren
  4. Speichern
Wo findest du Subscriptions? Oben rechts auf deinen Benutzernamen klicken → Subscriptions im Dropdown-Menü.

User als RM & PM (API-Script)

⏱ ca. 5 Minuten

📚 Theorie: Automatisierung

Das Script setup_tempus_workflows_manuel.py legt den User an und verbindet ihn mit Stern & Co Ressourcen und Projekten via Tempus Supergrid API.

🛠 Verwendung

cd "Onboarding Valkeen/scripts"
export TEMPUS_BASE_URL="https://trial5.tempus-resource.com/slot4"
export TEMPUS_API_KEY="<dein-api-key>"

# 1. Zuerst: Custom Fields der Instanz prüfen
python setup_tempus_workflows_manuel.py --explore

# 2. User anlegen und verbinden
python setup_tempus_workflows_manuel.py --setup

Falls Custom-Field-Namen abweichen: Script anpassen oder manuell in der UI einrichten.

Excel-Alternative

⏱ ca. 5 Minuten

📚 Theorie: Resources-Excel mit RM

Das Script generate_stern_co_excel_import.py erzeugt eine Resources-Excel mit:

  • Resource Managers = User (für alle Stern & Co Ressourcen)
  • User als zusätzliche Ressource mit Is Resource Manager = TRUE

🛠 Verwendung

cd "Onboarding Valkeen/scripts"
pip install openpyxl
python generate_stern_co_excel_import.py

Erzeugt docs/Stern_Co_Resources.xlsx. Importieren via Tempus Excel Data Sync → Resources.

Prüfen & Testen

⏱ ca. 5 Minuten

🛠 Checkliste

  1. Als User einloggen
  2. Resource Management → Quick Filter «My resources» → Stern & Co Ressourcen sichtbar?
  3. Resource Requests → Pending-Anfragen sichtbar? (Falls PM eine erstellt hat)
  4. Subscriptions: Resource Request → Submitted aktiv?
Wo findest du «My resources»? Resource Management → oben im Grid → Quick-Filter-Dropdown (oft neben «All resources») → «My resources» wählen.

Workflow-Einrichtung abgeschlossen. Viel Erfolg!
