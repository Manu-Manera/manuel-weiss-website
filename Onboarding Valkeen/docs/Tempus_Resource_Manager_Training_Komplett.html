Kapitel 0: Übersicht & Shortcuts

⏱ ca. 5 Minuten

📚 Theorie: Übersicht

  • User Access: Tempus-Zugang variiert je nach Setup. Ressourcen loggen sich via SSO ein.
  • Ressourcentypen: Named Resources vs. Demand Planning (Platzhalter).
  • Projekttypen: Standard, Opportunities, Non-Project Activities (Absence, Admin, Run, Change).
  • Assignments: Verknüpfung Ressource ↔ Projekt. Zeit-Einheiten: Hours, FTE, Mandays.
  • Resource Request Workflow: RM müssen Assignments genehmigen (Approve, Replace, Reject).

📚 Theorie: Tempus Shortcuts

  • Spalten sortieren: Klick auf Spaltenkopf
  • Kopieren: Ctrl+C / Ctrl+V
  • Icons: 🔴 Papierkorb = Löschen · ✏️ Stift = Bearbeiten · 🔒 = Gesperrt · 🔔 = Benachrichtigung
Wo findest du was? Nach dem Login: 6 Tiles auf der Homepage (Project Management, Resource Management, Resource Requests, BPA Flatgrid, Report Management, Bulk Resource Capacities). Task Bar oben: Tempus-Logo links, Glocke, Hamburger-Menü, Username rechts.

🛠 Praxis: Stern & Co – Erste Schritte

  1. Öffne Tempus Resource in Chrome.
  2. Klicke auf Healthineers-ID Login und melde dich an.
  3. Identifiziere alle 6 Tiles auf der Homepage.

Workflow-Einrichtung

⏱ ca. 15 Minuten

📚 Theorie: Resource Request Workflow

Der Workflow ist in Tempus eingebaut: PM erstellt Request → Status Pending → RM erhält Benachrichtigung → RM Approve / Replace / Reject → Status wechselt zu Approved/Rejected.

Voraussetzung: Ressourcen müssen einen Resource Manager zugewiesen haben. Der RM muss Is Resource Manager = true haben.

📚 Theorie: Rollen & Attribute

AttributWoBedeutung
Resource ManagersResource → Attributes & IdentityWer managed diese Ressource?
Is Resource ManagerResource → Attributes & IdentityIst diese Person ein RM?
Require Resource Manager ApprovalResource → Attributes & IdentityMuss RM Assignments genehmigen?
Project ManagerProjekt → AttributesWer ist PM für dieses Projekt?
Wo findest du die Attribute? Resource: Resource Management → Ressource öffnen → Tab «Attributes & Identity» → SHS IT Attributes (oder vergleichbar). Projekt: Project Management → Projekt öffnen → Attributes.

🛠 Praxis: Manuel als RM & PM einrichten

  1. Manuel anlegen (falls nicht vorhanden): Resource Management → Plus → Name «Manuel» → Create.
  2. Manuel als RM: Manuel öffnen → Attributes & Identity → Is Resource Manager = Ja.
  3. Stern & Co Ressourcen: Jede Ressource (Brad Pitt, Meryl Streep, …) → Resource Managers = Manuel, Require Resource Manager Approval = Ja.
  4. Stern & Co Projekte: Jedes Projekt → Project Manager = Manuel (falls Attribut vorhanden).
  5. Subscriptions: Als Manuel einloggen → Username → Subscriptions → Resource Request → Submitted aktivieren.
